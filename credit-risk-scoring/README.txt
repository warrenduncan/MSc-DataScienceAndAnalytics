##################################################################################################################
############################################### CREDIT SCORECARDS ################################################
##################################################################################################################

This analysis considered credit data, and created models from the data that could be used in application scoring.
The output of each model is simply to determine the likelihood of a 'Good' or 'Bad' debtor at the point of 
application.

The analysis uses linear regression and logistical regression, and is completed using the R programming language.


##################################################################################################################

FILE DESCRIPTIONS:
 	* CreditData_code.ipynb: A jupyter notebook containing all the code for the analysis, which was completed 
				 using the R programming language. 
 	* CreditData.xlsx: An excel spreadsheet containing the data used in the analysis, which is also freely available on Kaggle
